# Erasmus Chatbot — UL FRI
### NLP Course Project | Group: hungover_pandas

A domain-specific RAG (Retrieval-Augmented Generation) chatbot for students at UL FRI who want to go on Erasmus exchange. The chatbot answers questions about partner universities, deadlines, financial support, visas, Learning Agreements, and more — grounded exclusively in official UL FRI Erasmus documents.

## How it works

```
INDEXING (run once)
  data/*.txt + data/*.pdf + data/**/*.txt
      └─► chunk into 500-char pieces (50 overlap)
              └─► embed with all-MiniLM-L6-v2
                      └─► store in FAISS index

QUERY (at runtime)
  User question
      └─► embed question
              └─► FAISS MMR retrieval (top 3 chunks)
                      └─► build prompt (system + context + question)
                              └─► Llama 3.1 / Gemma 3 (4-bit on GPU)
                                      └─► answer grounded in documents
```

## Running the pipeline

### Step 1 — Build the index
Run this once, or again whenever you add/update files in `data/`:
```bash
python code/index.py
```

### Step 2 — Test the LLM (optional but recommended)
Verifies the model loads and generates text before running the full pipeline:
```bash
python code/llm_test.py
# or for Gemma 3:
python code/llm_test.py --model models/gemma-3-4b-it
```

### Step 3 — Run the chatbot

**Interactive chat:**
```bash
python code/rag.py
```
Type your questions. Special commands:
- `debug` — toggle source display (shows which documents were retrieved)
- `quit` — exit

**Predefined test questions:**
```bash
python code/rag.py --mode test
```

**With a different model:**
```bash
python code/rag.py --model models/gemma-3-4b-it
```

### Step 4 — Evaluate
Runs all 210 QA pairs through the RAG and saves results:
```bash
# Quick test (first 10 questions)
python code/evaluate.py --limit 10

# Full evaluation
python code/evaluate.py

# Evaluate Gemma 3 separately
python code/evaluate.py \
  --model models/gemma-3-4b-it \
  --output evaluation/results_gemma3.csv
```
Results are saved to `evaluation/results.csv` and `evaluation/summary.txt`.

## Models

| Model | Path | Size | Notes |
|-------|------|------|-------|
| Llama 3.1-8B-Instruct | `models/llama-3.1-8b-instruct` | ~16 GB | Default, best quality |
| Gemma 3-4B-IT | `models/gemma-3-4b-it` | ~8 GB | Faster, good for comparison |

Both models use **4-bit NF4 quantisation** on CC >= 8.0 GPUs (H100, A100), which reduces VRAM usage significantly while maintaining quality.

---

## Evaluation metrics

| Metric | Description |
|--------|-------------|
| **Token F1** | Word overlap between predicted and expected answer. Low scores are expected due to paraphrasing — this is normal for generative RAG. |
| **Key info hit rate** | Whether the answer contains important terms from the ground truth. More meaningful than F1 for this use case. |
| **Fallback rate** | How often the model says "I don't have that information." Indicates gaps in the knowledge base. |

## SLURM batch job (non-interactive)
If you want to run evaluation overnight without staying connected:
```bash
sbatch logs/run_rag.sh
# Check status:
squeue -u $USER
# Check output:
tail -f logs/rag_<JOBID>.out
```
